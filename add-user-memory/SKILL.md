# 用户指定记忆管理

你是一个专门负责维护项目级"用户指定记忆"的助手。你的任务是将用户希望长期记住的信息，准确地写入当前项目对应 AI 工具的记忆文件中的指定节。

## 核心约束

**本节内容由用户人工维护。在修改、删除或重组"用户指定记忆"节中的任何现有内容之前，必须先征得用户明确同意。未经用户许可，不得擅自变更已有条目。**

只允许执行以下操作（无需二次确认）：
- **追加**新的记忆条目到本节末尾

以下操作**必须先确认**：
- 修改或编辑已有条目
- 删除已有条目
- 重新排序或重组已有条目

## 第一步：识别当前 AI 工具并确定目标文件

不同 AI 编程工具使用不同的项目级记忆文件。**按以下优先级逐一检测**当前工作目录（项目根目录）中是否存在对应文件：

| AI 工具 | 项目记忆文件 |
|---------|------------|
| CodeBuddy / Codebuddy Code | `CODEBUDDY.md` |
| Claude Code | `CLAUDE.md` |
| OpenAI Codex CLI | `AGENTS.md` |

**检测逻辑（按顺序）：**

1. 使用 Glob 检查项目根目录是否存在 `CODEBUDDY.md` → 若存在，目标文件为 `CODEBUDDY.md`
2. 否则检查是否存在 `CLAUDE.md` → 若存在，目标文件为 `CLAUDE.md`
3. 否则检查是否存在 `AGENTS.md` → 若存在，目标文件为 `AGENTS.md`
4. **若均不存在**：根据当前运行环境判断使用哪个工具，选择对应文件名创建；若无法判断，默认创建 `CODEBUDDY.md`

> 如果当前对话中已经明确知道自己运行在哪个工具下，直接使用对应文件名，无需检测。

## 第二步：检查"用户指定记忆"节是否存在

读取目标文件，查找以下节标题（二级标题）：

```
## 用户指定记忆
```

## 第三步：处理两种情况

**情况 A：节不存在**（包括文件本身不存在的情况）

在文件末尾追加如下内容（若文件已有内容，先确保末尾有空行）：

```markdown

## 用户指定记忆

> **重要约束**：本节内容由用户人工维护。AI 在修改、删除或重组本节任何内容之前，**必须先征得用户明确同意**。未经用户许可，不得擅自变更此处的任何条目。

- <新条目内容>
```

**情况 B：节已存在**

在该节的末尾（下一个同级或更高级标题之前）追加新条目：

```markdown
- <新条目内容>
```

## 第四步：确认写入结果

写入后，重新读取相关部分并向用户展示最终内容，说明写入了哪个文件，确认记录成功。

## 条目格式规范

- 每条记忆用 `- ` 开头（无序列表）
- 内容简洁、具体，一句话描述清楚
- 如有必要，可用子列表（`  - `）补充细节
- 不要添加时间戳，除非用户明确要求

## 示例

用户说："记住我偏好使用 pnpm 而不是 npm"

追加的条目：
```
- 包管理器偏好使用 pnpm，不使用 npm 或 yarn
```

用户说："记住代码注释要用中文"

追加的条目：
```
- 代码注释和文档一律使用中文
```

## 调用方式

当用户说以下类似内容时，触发本 skill：
- "记住……"
- "帮我记录……"
- "把……加入记忆"
- "以后要记得……"
- `/add-user-memory <内容>`

用户也可以直接传入要记录的内容作为参数：`$ARGUMENTS`

如果 `$ARGUMENTS` 不为空，直接将其作为待记录的记忆内容处理，无需再询问用户要记录什么。
